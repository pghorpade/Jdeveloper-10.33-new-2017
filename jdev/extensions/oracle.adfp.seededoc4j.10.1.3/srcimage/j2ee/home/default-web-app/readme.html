WebCenter Preconfigured OC4J

For Use with Oracle JDeveloper 10.1.3.2.0 and above

Last Updated: April 10, 2007
Status: Production
Version: 10.1.3.3.0

Introduction

This extension is the WebCenter preconfigured OC4J. This OC4J contains the following components to facilitate developing WebCenter applications with JDeveloper:

  • JSR 168 portlet container (exposing portlets through WSRP 1.0 and a preliminary version of WSRP 2.0)
  • PDK-Java runtime libraries
  • ADF and WebCenter runtime libraries
  • OmniPortlet
  • Web Clipping
  • Rich Text Portlet
  • WSRP Sample Portlets
  • PDK-Java Sample Portlets

To open this readme page again, select menu item Help -> WebCenter Preconfigured OC4J Readme.

Managing the Preconfigured OC4J

This extension sets up menu items and corresponding toolbar icons in JDeveloper:

  • Start WebCenter Preconfigured OC4J:
  • Stop WebCenter Preconfigured OC4J:

Select the menu items (Tools > Start/Stop WebCenter Preconfigured OC4J) or click the toolbar icons to start or stop the preconfigured OC4J.

Preconfigured OC4J Installation

Starting the preconfigured OC4J for the first time completes its installation.

If you have not already set the $JDEV_USER_DIR environment variable, the preconfigured OC4J installs in the $ORACLE_HOME/jdev/extensions/oracle.adfp.seededoc4j.10.1.3 directory on Windows. On UNIX platforms, It installs in the $HOME/jdevhome/extensions/oracle.adfp.seededoc4j.10.1.3 directory.

If you have already set the $JDEV_USER_DIR environment variable, users who share the same Oracle JDeveloper installation but have different $JDEV_USER_DIR directories will have an individually preconfigured OC4J installation. In this case, the preconfigured OC4J installs in the $JDEV_USER_DIR/extensions/oracle.adfp.seededoc4j.10.1.3 directory.

Index Page

After you start the preconfigured OC4J, confirm that it has started by waiting for the Oracle Containers for J2EE 10g (10.1.3.3.0) initialized message in the Log window in Oracle JDeveloper.

Once OC4J starts, bring up the index page to test OC4J. The index page contains a list of links to the preconfigured portlet producers. You can test each of the portlet producers by clicking the links.

Default Environment Settings

Default Password for oc4jadmin

The default password of the oc4jadmin user is welcome.

Default Ports

The preconfigured OC4J uses the following port numbers:

  • HTTP: 6688
  • RMI: 22667
  • RMI SSL: 22668
  • JMS: 9168

If these ports conflict with others, you can change the ports of your preconfigured OC4J. For information, refer to the OC4J Configuration and Administration Guide at $ORACLE_HOME/j2ee/home/default-web-app/oc4jusersguide.pdf.

If you change the RMI port, enter the new port number in the command line to stop OC4J. See the "Cannot Find Java" section under the Troubleshooting section in this document.

Creating a Connection to the Preconfigured OC4J

Assuming you are using the default settings, follow these steps to create a connection to your preconfigured OC4J:

Step 1:
Connection Name: LocalOC4J [or any name you like]
Connection Type: Standalone OC4J 10g 10.1.3

Step 2:
Username: oc4jadmin
Password: welcome
Deploy password: [ ] [optional]

Step 3:
Host Name: localhost
RMI Port: 22667
URL Path: [leave empty]

Migration

When you install a new version of JDeveloper and select to migrate settings from a previous release, JDeveloper will detect if you have Preconfigured WebCenter OC4J in previous installation. Once detected, the portlet data of the preconfigured portlets in $ORACLE_HOME/portal/portletdata will be migrated automatically.

You still need to migrate the other settings such as proxy information of OmniPortlet and Web Clipping yourself. You also need to migrate PDK-Java Sample Portlet data manually. PDK-Java Sample Portlet data is in $ORACLE_HOME/j2ee/home/applications/jpdk/jpdk/WEB-INF/providers/<provider name>.

For custom portlets, you need to deploy them to Preconfigured OC4J again and copy over the portlet customization data from previous Preconfigured OC4J (if the customization data is not in $ORACLE_HOME/portal/portletdata).

Troubleshooting

Executable Privilege of Command Scripts

On some platforms such as Mac OS, you must manually set executable privilege on the following command script that starts OC4J:

$JDEV_USER_DIR/extensions/oracle.adfp.seededoc4j.10.1.3/bin/adfp_oc4j.sh

Cannot Find Java

The Start/Stop WebCenter PreConfigured OC4J menu items and toolbar icons are included within Oracle JDeveloper External Tools. In some environments, the extension cannot determine $JAVA_HOME and $ORACLE_HOME to construct the correct command line for the external tools.

To resolve this issue, you can manually edit the external tools and enter correct $JAVA_HOME and $ORACLE_HOME in the command line.

To edit the external tool definition, select menu item Tool -> External Tools.

The command line to start the preconfigured OC4J is:

adfp_oc4j.cmd java_home oracle_home ext_dir ?start (on Windows)

adfp_oc4j.sh java_home oracle_home ext_dir ?start (on other platforms)

You can also use the command line to stop the preconfigured OC4J by using the following commands:

adfp_oc4j.cmd java_home oracle_home ext_dir -shutdown -port rmi.port -password welcome (on Windows)

adfp_oc4j.sh java_home oracle_home ext_dir -shutdown -port rmi.port -password welcome (on other platforms)

where ext_dir is the preconfigured OC4J installation directory:

$JDEV_USER_DIR/extensions/oracle.adfp.seededoc4j.10.1.3
